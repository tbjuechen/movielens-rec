# MovieLens 32M 数据探索性分析 (EDA) 报告

## 1. 数据概览
- **评分记录数**: 32,000,204 条
- **用户数**: 200,948 名
- **电影数**: 87,585 部 (其中 84,432 部有评分记录)

## 2. 关键发现与统计指标

### 2.1 极高的稀疏度 (Sparsity)
- **数值**: **99.8114%**
- **分析**: 用户-物品矩阵极其稀疏，绝大多数用户只看过了极小比例的电影。
- **指导意义**: 
    - 传统的协同过滤模型（如协同矩阵分解）面临巨大的挑战。
    - 必须引入电影元数据（题材、年份、标签）等内容特征来辅助推荐。

### 2.2 显著的长尾效应 (Long-tail Effect)
- **数值**: **前 10% 的热门电影贡献了约 95.70% 的评分量。**
- **分析**: 流量高度集中在少数热门影片中。
- **指导意义**:
    - **热门召回 (Popularity-based)** 效果会非常稳健，是必须建立的基准路。
    - 需要在排序阶段引入多样性策略，否则用户将陷入“信息茧房”。

### 2.3 用户行为分布
- **活跃度**: 中位数评分数 (73) 远低于平均值 (159.2)。
- **极端值**: 存在打分量过万的“超级用户”。
- **指导意义**: 训练模型时需对超级活跃用户进行降权或采样，防止其偏好主导全局模型。

### 2.4 评分值偏置
- **分布**: 评分主要集中在 **3.0 - 5.0** 之间，高分居多。
- **分析**: 用户倾向于给看过的电影打好分，或者只对自己感兴趣的电影打分。
- **指导意义**:
    - 将回归问题转化为**二分类问题**（CTR 预估）可能更符合业务逻辑。
    - 建议以 3.5 分或 4.0 分作为正负样本的分界线。

### 2.5 题材分布
- **主流题材**: Drama (剧情) 和 Comedy (喜剧) 占据绝对优势。
- **指导意义**: 题材特征是进行内容召回（Content Recall）最可靠的标签。

---

## 3. 算法设计初步规划 (基于 EDA)

1. **召回阶段 (Multi-channel Recall)**:
    - **热门路**: 召回全站热度最高的电影。
    - **内容路**: 根据用户历史喜好题材，召回同题材下的高分电影。
    - **协同路**: 利用 ItemCF 或向量化方法寻找相似项。

2. **排序阶段 (Ranking)**:
    - 消除用户打分偏置（User Bias）。
    - 引入时间衰减特征（近期评分权重更高）。

3. **策略阶段 (Rerank)**:
    - 针对长尾电影进行一定的曝光扶持。
    - 强制过滤掉用户已观看过的电影。
